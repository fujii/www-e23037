CSSの問題点

HTMLで記述した内容におめかしして綺麗にみせるのがスタイルシートの役目です。 CSS ( Cascading Style Sheets ) とはスタイルシートのひとつです。 CSSを利用すれば文字の色を指定することができます。 色の指定の仕方は大きく2通りあります。 ひとつは #f00 のように光の三原色である赤、緑、青を値で指定するものです。 ちなみに #f00 は赤を意味します。 もう一つは Background のように予約された識別子で指定する方法です。 ちなみに Background はデスクトップの背景色です。

背景色を指定しないページにとって、CSS2は機能が不十分です。 例えば、以下ようなとき。 文章は透明なボーダーと呼ばれる長方形で四方を囲まれています。 このボーダーの線に太さを指定し、色を指定することにより、 その文章を強調することができます。 色の指定は強調するのだから、少なくとも目に見えなくてはなりません。 つまり、透明色と背景色以外ということです。 #000 と指定したとします。 これで確実に黒い線を引くことができます。 しかしながら、これでは背景色も黒のときに目に見えないのです。 文字色と同じ色を示す予約された識別子を使用すればよいわけです。 しかしながら、それが現時点でCSSに存在しないのです。 追加の必要な予約語は6つ。 body要素で指定可能な属性 text 、 link 、 vlink 、 alink 、bgcolor のデフォルト値、 そして、透明色です。

実はもっとよい実現方法も考えられます。 

h1{
 border-color: body->color;
}

このような感じで body の文字色の値を参照し、 ボーダーの色に代入する。

CSSの規格を作っている方がいらっしゃいましたら、 CSS3のときにはよろしくお願いします。
